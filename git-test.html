20 Preguntas sobre Git

Git es un sistema de control de versiones distribuido que permite el seguimiento de cambios en archivos y coordinación de trabajo en proyectos colaborativos.

Git es el sistema de control de versiones, mientras que GitHub es una plataforma en línea que proporciona servicios de alojamiento de repositorios Git y herramientas de colaboración adicionales.

El comando es git init. Se ejecuta en el directorio raíz del proyecto para crear un nuevo repositorio Git en esa ubicación.

El comando es git clone <URL>, donde <URL> es la ubicación del repositorio Git remoto. Esto descargará una copia completa del repositorio en tu máquina local.

Un repositorio Git local es una copia del repositorio en tu máquina local, mientras que un repositorio remoto se encuentra en un servidor remoto, como GitHub. Los cambios se pueden sincronizar entre ambos.

El comando git status muestra el estado de los archivos en tu repositorio local, incluyendo los archivos modificados, los archivos nuevos y los archivos eliminados.

El comando git add <archivo> agrega un archivo específico al área de preparación. Para agregar todos los cambios, se puede usar 'git add .'

El comando es git commit -m "mensaje del commit". Esto guarda los cambios confirmados en el repositorio local con un mensaje descriptivo.

El comando git checkout -- <archivo> descarta los cambios no confirmados en un archivo específico. Para descartar todos los cambios no confirmados, se puede usar git checkout ..

El comando es git log. Muestra una lista de todas las confirmaciones realizadas en el repositorio, junto con detalles como el autor, la fecha y el mensaje del commit.

El comando git branch <nombre_de_rama> crea una nueva rama con el nombre especificado en el repositorio local.

El comando es git checkout <nombre_de_rama>. Te permite cambiar a la rama especificada para continuar trabajando en ella.

Primero, cambia a la rama de destino con git checkout <rama_destino>. Luego, ejecuta git merge <rama_fuente> para fusionar la rama fuente en la rama de destino.

La fusión combina los cambios de dos ramas diferentes y crea un nuevo commit de fusión, mientras que el rebase mueve los cambios de una rama a otra, reescribiendo la historia del repositorio.

Los conflictos de fusión ocurren cuando hay cambios conflictivos en archivos que se están fusionando. Para solucionarlos, se deben editar los archivos manualmente, eliminando las secciones en conflicto, y luego confirmar los cambios.

El comando es git push <nombre_remoto> <nombre_rama>. Esto envía los cambios locales de la rama especificada al repositorio remoto.

El comando git pull <nombre_remoto> <nombre_rama> obtiene los últimos cambios del repositorio remoto y los fusiona automáticamente con tu repositorio local.

Un conflicto de fusión ocurre cuando hay cambios conflictivos en archivos que se están fusionando. Esto sucede cuando dos ramas tienen cambios diferentes en la misma sección del mismo archivo.

El comando git branch -d <nombre_rama< elimina la rama especificada del repositorio local. Para eliminar una rama remota, se puede usar git push <nombre_remoto< --delete <nombre_rama<.

El comando git revert HEAD deshace el último commit, creando un nuevo commit que revierte los cambios introducidos en el último commit. Esto permite mantener un historial limpio sin perder cambios anteriores.
